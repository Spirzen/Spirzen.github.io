Роль автора

Материалы пишутся с 2018 года; публичная платформа — с 2025. Я — автор, методист, Lead Developer и Content Architect: от модели знаний и трёхрепозиторной архитектуры до iframe-интеграции, DocSearch, CI/CD и ~500 React-демо на play.spirzen.ru.

Проект закрывает потребность в едином верифицированном источнике IT-знаний: encyclopedia как хаб, code/play как специализированные витрины, cross-origin embed без backend. Бесплатно, без рекламы и сбора данных.

Архитектура экосистемы

Три независимых репозитория → три домена на GitHub Pages → интеграция через iframe/postMessage. Read-only SSG без backend: Markdown в Git → сборка → статический build/ / dist/. Cloudflare (DNS, CDN, WAF, SSL) перед Pages.

  1. Экосистема (C4) Автор → 3 репозитория GitHub → GitHub Actions на каждый домен → Cloudflare → spirzen.ru / code.spirzen.ru / play.spirzen.ru.
  2. it-knowledge-base docs/ (~3400), src/ (ExternalCodeEmbed, ExternalPlayEmbed, DocSearch), препроцессоры wiki-links и search-index → Docusaurus 3.10.
  3. it-code-examples examples/ (~2312), Astro 5 + Shiki, каталог, серии, embed /e/embed/ → dist/.
  4. it-play plays/ (~500), Astro + React 19 islands, demoRegistry, embed /p/embed/ → dist/.
  5. Runtime Статья на spirzen.ru → iframe code/play + postMessage (тема, высота, fullscreen) + лёгкие inline-виджеты; MAUI WebView → spirzen.ru.
